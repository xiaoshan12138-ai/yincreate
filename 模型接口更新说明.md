# 模型查询接口更新说明

## 📋 更新内容

### 接口地址
```
GET /api/v1/models
```

### 更新时间
2026-05-15

## ✅ 已实现

### 1. 兼容 Token Switch 模型

**之前**: 只返回腾讯云模型
**现在**: 同时返回腾讯云和 Token Switch 模型

### 2. 新增字段

每个模型现在包含以下字段：
```json
{
  "id": "模型ID",
  "name": "显示名称",
  "description": "描述",
  "vendor": "vendor_a 或 vendor_b",
  "vendor_name": "腾讯云 VOD 或 Token Switch",
  "is_new": false,
  "is_vip": false
}
```

### 3. 新增统计信息

响应中新增 `summary` 字段：
```json
{
  "summary": {
    "total_image_models": 11,
    "total_video_models": 21,
    "total_voices": 9,
    "vendors": ["vendor_a", "vendor_b"]
  }
}
```

## 🎯 前端使用建议

### 1. 按厂商分组显示

```javascript
fetch('http://192.168.31.243:8003/api/v1/models')
  .then(res => res.json())
  .then(data => {
    const models = data.data.image_models;
    
    // 按厂商分组
    const vendorA = models.filter(m => m.vendor === 'vendor_a');
    const vendorB = models.filter(m => m.vendor === 'vendor_b');
    
    console.log('腾讯云模型:', vendorA);
    console.log('Token Switch 模型:', vendorB);
  });
```

### 2. 显示厂商标签

```html
<select id="model-select">
  <optgroup label="腾讯云 VOD">
    <option value="kling_3_0" data-vendor="vendor_a">Kling 3.0</option>
  </optgroup>
  <optgroup label="Token Switch">
    <option value="gpt-image-2" data-vendor="vendor_b">GPT Image 2</option>
  </optgroup>
</select>
```

### 3. 根据选择的模型自动设置 vendor

```javascript
const modelSelect = document.getElementById('model-select');
const selectedOption = modelSelect.options[modelSelect.selectedIndex];
const vendor = selectedOption.dataset.vendor;

// 在生成请求中使用
const request = {
  output_type: 'image',
  model: selectedOption.value,
  vendor: vendor,  // 自动设置
  prompt: '...'
};
```

## 📝 注意事项

1. **vendor 字段必需**: 前端在调用生成接口时，需要根据选择的模型设置正确的 `vendor` 参数

2. **模型 ID 格式不同**:
   - 腾讯云: `kling_3_0` (下划线分隔)
   - Token Switch: `gpt-image-2` (连字符分隔)

3. **模型名称使用**:
   - 在生成接口中使用 `name` 字段（完整名称）
   - 例如: `"model": "Kling 3.0"` 或 `"model": "gpt-image-2"`

4. **兼容性**: 旧的前端代码仍然可以工作，只是不会显示 Token Switch 模型

## 🔄 迁移指南
